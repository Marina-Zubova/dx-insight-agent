The Problem

Developer experience research is manual, slow, and anecdotal

📊 How it works today

Read PRs manually
Run surveys (low response)
Guess at friction points
Report once, forget

⚠️ What we miss

Patterns across hundreds of PRs
Real-time friction signals
Evidence-backed recommendations
Continuous monitoring

The Idea

What if an AI agent continuously observes pull requests
and surfaces friction patterns?

🔍

Observe

Read PRs, reviews,
timelines, CI status

🧠

Analyze

Identify friction, categorize themes

📋

Report

Evidence-backed findings,
confidence scores

Not a code reviewer. A UX researcher.

How It Works

OpenClaw Agent + GitHub API + Heartbeat Scheduler


# HEARTBEAT.md — the agent's schedule
tasks:
  - name: dx-analyze-prs
    interval: 6h
    prompt: "Find newly merged PRs, produce friction reports"

  - name: dx-weekly-summary
    interval: 24h
    prompt: "Aggregate patterns, generate repo DX summary"

Every 6 hours: scan → collect evidence → analyze → report
Every 24 hours: aggregate → identify recurring patterns

Research Rigor

Not just pattern-matching — self-correcting analysis

❌ Removed (false positives)

"Pipeline broken at launch" — the fix was proactive, not reactive
"Community users asking for ETAs" — demand, not friction
"Merge delay after approval" — 3.5 hours is trivial

✅ Added (missed friction)

Test infrastructure as onboarding barrier
Silent breaking change in library internals
Contributor consulting ChatGPT to debug framework tests

Confidence scores Evidence quotes Gaps acknowledged Self-critique
